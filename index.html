Claude Design を
ゼロから使えるようになる

チャットで会話しながら、デザイン・プロトタイプ・資料を作る Claude の新機能。専門知識ゼロでも、この順番どおりに読めば最初の1枚が作れます。

📍 対象：はじめて触る人 ⏱ 読了の目安：約15分 🔗 公式：claude.ai/design

1Claude Design とは？

「絵を描くソフト」ではなく「お願いするとデザインしてくれる相棒」です。

Claude Design（クロード・デザイン）は、Anthropic（アンソロピック／Claude を作っている会社）が2026年4月に公開したデザイン機能です。マウスでパーツを並べて作るのではなく、「こういうものが欲しい」と言葉で伝えると、Claude が画面を作ってくれます。

こんなものが作れます

Webサイト・LP（ランディングページ） アプリの画面イメージ 動くプロトタイプ プレゼン資料・ピッチデック ダッシュボード バナー・販促物

画面は左がチャット欄、右が「キャンバス」（作品が表示される場所）の2つに分かれています。あなたが左で話しかけると、右にデザインが現れ、会話を重ねるたびに少しずつ理想に近づいていきます。

💡 一言でいうと「Photoshop や Figma の使い方を覚えなくても、日本語でお願いするだけでデザインができる」ツールです。

2まず覚える用語

この5語だけ知っておけば、あとの説明がスッと入ります。タップで開きます。

キャンバス（Canvas）
作ったデザインが表示される、画面の右側の作業スペースのこと。紙やホワイトボードのようなもの。ここに完成イメージが出ます。
プロンプト（Prompt）
Claude にお願いする「指示文」のこと。「ヒーローセクション付きのLPを作って」のような一文。良いプロンプトの書き方は13章で解説します。
プロトタイプ（Prototype）
実際に動く「試作品」。ボタンを押すと画面が切り替わる、など本物に近い動きを確認できる状態。発表前のテストに便利。
デザインシステム（Design System）
会社やチームの「色・フォント・ボタンの形」などのルール集。これを読み込ませると、作るデザインが自動でブランドの見た目に揃います（8章）。
エクスポート（Export）／ハンドオフ（Handoff）
エクスポート＝書き出し。作ったデザインを PDF・PowerPoint・HTML などの形で取り出すこと。ハンドオフ＝引き継ぎ。デザインを開発（コード化）の段階へ渡すこと（11章）。

3使える条件・始め方

「誰が使えるか」と「どこから入るか」を確認しましょう。

項目内容
利用できるプランPro / Max / Team / Enterprise（有料プラン）。現在はベータ版（試験公開）です。無料プランでは使えません。
入口（アクセス方法）① ブラウザで claude.ai/design を開く
② または Claude デスクトップアプリのサイドバーから
使える端末パソコン（Web・デスクトップ）のみ。スマホ非対応です。
料金の考え方チャットや Claude Code と共通の利用枠から消費されます。Design 専用の追加料金はありません。大きく複雑なものほど多く消費します。
Enterprise の注意初期設定ではオフになっています。管理者に有効化を依頼してください。
はじめの一歩有料プランにログインした状態で claude.ai/design を開く → 新しいプロジェクトを作る。これだけで準備完了です。

4最初のデザインを作る

いちばん大事な「基本の流れ」。実はたった4ステップです。

  1. 新しいプロジェクトを作るチームのデザインシステムが設定済みなら、色やフォントが自動で引き継がれます。何もなくてもOK、そのまま進めます。
  2. 欲しいものを言葉で伝える（プロンプト）「何を・どんな配置で・何を載せて・誰向けか」を一文で。例は下の枠を「コピー」して使ってみてください。
  3. キャンバスの結果を見る右側に最初のデザインが現れます。完璧でなくてOK。ここから育てていきます。
  4. 会話や直接編集で仕上げる「もっと暗い配色に」「ここを大きく」と伝えるたびに改善されます（詳しくは6章）。

そのまま使えるプロンプト例

例1：ランディングページ

新しいAI翻訳アプリのランディングページを作って。ヒーローセクション（キャッチコピー＋登録ボタン）、機能を3つ並べた紹介、料金プラン、お客様の声、フッターを縦に配置して。明るく信頼感のある配色で、対象は中小企業の担当者。

例2：ダッシュボード

月間の売上を表示するダッシュボードを作って。上段に「売上・新規顧客・解約率」の3つの数値カード、その下に月別売上の折れ線グラフ、右側に地域でフィルターできる絞り込み。ビジネス向けの落ち着いた見た目で。

例3：アプリのオンボーディング

スマホアプリの初回登録フローを4画面で作って。①ようこそ ②メール登録 ③興味のあるカテゴリ選択 ④完了。各画面に進むボタンを付けて、若い世代向けの親しみやすいデザインで。

💡 ポイント最初から完璧を狙わず、シンプルに頼んで→少しずつ足すのが成功のコツ。迷ったら「2〜3パターン見せて」と頼むと選べます。

5画面（キャンバス）の見方

どこに何があるか分かれば、もう迷いません。

場所役割
左：チャット欄Claude に指示を出す場所。大きな変更や「やり直し」「別案」はここで頼む。
右：キャンバスデザインが表示される場所。要素をドラッグで移動・サイズ変更・整列でき、クリックしてコメントも付けられる。
右上：Export ボタン完成したデザインを書き出す・他ツールへ送る入口（10章）。
右上：Share ボタンリンクを作って人に共有（12章）。

つまり「ざっくりした変更は左のチャット」「細かい位置や見た目は右のキャンバスで直接」と覚えると迷いません。

63つの修正方法を使い分ける

直し方は3通り。場面で使い分けると一気に上達します。

方法得意なこと言い方の例
① チャット
（大きな変更）		全体の雰囲気・構成・別案づくり「配色を暗めでミニマルに」「指標を上の行にまとめて」「別レイアウトを2案見せて」
② インラインコメント
（ピンポイント）		特定の部品だけを直す要素をクリックして「このボタンの余白を広く」「ここをドロップダウンに」「ここはブランド色で」
③ 直接編集
（見た目の微調整）		位置・大きさ・整列をその場でマウスでドラッグ・リサイズ・整列。文字も直接打ち替え可能。
⚠ ベータ版の小ワザインラインコメントがまれに消える・反映されないことがあります。そのときは同じ内容をチャットに貼り付けて伝えれば確実です。
💡 上達のコツ「なんか違う」ではなく「余白を8pxに詰めて」「文字を1段大きく」のように具体的な数値・部品名で伝えると、思いどおりに直ります。

7参考資料を渡してブレを減らす

「こんな感じ」を見せると、一発で精度が上がります。

プロジェクトには、いつでも参考になる素材を追加できます。Claude はそれを見て方向性を合わせてくれます。

  • ビジュアル参考：スクリーンショット、競合のデザイン、手描きのワイヤーフレーム、インスピレーション画像
  • ドキュメント：Word（DOCX）、PowerPoint（PPTX）、Excel（XLSX）などの資料
  • コード／ファイル：既存のコードリポジトリを連携して、部品の構成に反映
💡 おすすめ言葉で説明しづらいときは、イメージに近い画像を1枚アップロードして「これくらいの雰囲気で」と添えるのが最短ルートです。

8デザインシステムと /design-sync

少し上級。会社のブランドに揃えたい人向けですが、概念だけ知っておけばOK。

デザインシステム＝色・フォント・ボタンの形などの「自社ルール集」。これを読み込ませると、作るものすべてが自動でブランドの見た目になります。

  1. セットアップ（デザイン責任者の作業）Claude が自社のコードやデザインファイルを読み取って、デザインシステムを自動で組み立てます。
  2. 適用は自動一度設定すれば、新しいプロジェクトに色・文字・部品パターンが勝手に反映されます。手動設定は不要。
  3. コードと同期したいときClaude Code（後述）で /design-sync と打つと、コードベースのデザインシステムを取り込めます。
用語メモリポジトリ（repository）＝プログラムのコードを保管している置き場のこと。「コードを連携」とは、この置き場をClaudeに見せることです。

9バージョンを残して試す

「今のを残したまま、別案も試したい」を叶える一言。

大きく方向転換したいときは、こう伝えるだけ。今の状態が保存され、安心して別案に挑戦できます。

バージョン保存の魔法の一文

今のデザインを保存して、まったく別のアプローチを試して。

Claude は現在のプロジェクトを保存し、どこに保存したかを教えてくれるので、あとから見比べられます。

11Claude Code へ引き継ぐ（ハンドオフ）

「デザイン → 本物のソフト（コード）」へ進める段階。あなたが選んだ目的地です。

デザインが固まったら、それを実際に動くソフトに変えるために Claude Code（Claude のコーディング担当）へ引き継げます。スクリーンショットから作り直すのではなく、あなたの作業の続きから開発が始まるのが特徴です。

引き継ぎ先こんな人に
ローカルのコーディングエージェント自分のPCで開発環境を持っている人
Claude Code Webブラウザだけで手軽に進めたい人
⚠ 初回の注意/design や /design-sync の機能は、Claude Code の新しいセッションで /update を一度実行してから使えるようになります。
流れのまとめ① Design で見た目を作る → ② 固まったら Claude Code へハンドオフ → ③ コードとして仕上げて公開。デザインと開発が地続きになります。

12共有する

右上の Share から、権限を選んでリンクを発行します。

権限できること
閲覧のみ（View）見るだけ。関係者への共有に
コメント（Comment）見て意見を書き込める。レビュー依頼に
編集（Edit）一緒に直せる。共同作業に
⚠ 範囲に注意共有は自分の組織（チーム）内に限られます。社外の人へはそのまま渡せません。また、複数人での同時編集はベータ版では不安定なことがあります。

13良いプロンプトのコツ

指示の質＝出来の質。この「4要素」を入れるだけで激変します。

要素意味例
🎯 目的何を作るか「採用ページを作って」
📐 レイアウトどう並べるか「上にヒーロー、下に職種一覧」
📝 内容何を載せるか「会社の魅力3つと応募ボタン」
👥 対象誰向けか「20代のエンジニア向け」

やってみよう（4要素入りの完成形）

4要素を全部入れた例

20代エンジニア向けの採用ページを作って。上にキャッチコピーと応募ボタンのヒーロー、その下に会社の魅力を3つカードで、最後に募集中の職種一覧。スタートアップらしい先進的でクリーンな配色で。

  • 最初はシンプルに頼み、あとから少しずつ足す
  • フィードバックは具体的に（「余白を8pxに」など数値で）
  • デザインシステムの部品名で指定する（「プライマリボタンを使って」）
  • 早めにレスポンシブ（スマホ・タブレット・PC）を意識
  • 迷ったら「2〜3案見せて」と頼む
  • 仕上げに「アクセシビリティと使いやすさをレビューして」と頼む

14困ったとき / 制限事項（ベータ版）

「壊れた？」と思う前に、まずここを確認。

コメントが消える・反映されない
ベータ版でたまに起きます。同じ内容をチャットに貼り付けて伝えれば確実に反映されます。コメント一覧の表示も確認してみてください。
大きなリポジトリで動作が重い
巨大なコードを直接読ませると遅くなることがあります。その場合は Claude Code 側から /design-sync を使うと安定します。
/design や /design-sync が使えない
Claude Code の新しいセッションで、最初に /update を実行する必要があります。
チャットがエラーになる
同じプロジェクト内で新しいタブを開き直すと回復することがあります。
スマホで使いたい
現在はWeb・デスクトップ専用で、モバイルには未対応です。
急に使えなくなった
利用枠（チャットなどと共通）を使い切った可能性があります。リセットまで待つか、クレジットを追加すると再び使えます。
デザインシステムの取り込みがいまいち
取り込みの品質は元コードの整理具合に左右されます。スタイル・フォント・部品が揃った状態で渡すと精度が上がります。

15練習：はじめてのチェックリスト

この順にチェックを付けながら、実際に1枚作ってみましょう。

  • 有料プランで claude.ai/design を開いた
  • 新しいプロジェクトを作った
  • 4要素（目的・レイアウト・内容・対象）入りのプロンプトを送った
  • キャンバスに最初のデザインが出た
  • チャットで全体を1回直した
  • 要素を1つクリックしてコメントで微調整した
  • 「2案見せて」で別案を比べた
  • Export から PDF か HTML で書き出した
  • Share でリンクを作って共有した
